# 🎀 공주비서 2.0 기획서 (확정본)

> 2026-07-15 확정. Discord 봇 1개 + LangGraph 멀티 에이전트 구조로,
> 8개 기록처(노션·텐미닛·옵시디언·티스토리·깃허브·블레이버스·인스타·오프라인타이머)를
> 비서의 팔다리로 연결한다.

---

## 1. 확정된 아키텍처 결정

| 항목 | 결정 |
|---|---|
| 봇 구조 | **Discord 봇 1개** + 내부 LangGraph Supervisor 멀티 에이전트. 응답은 기능별 웹훅 페르소나(📔기록요원, ⏱️시간요원, 📢발행요원, 🧠기억요원)로 구분 |
| 대화 기억 | LangGraph `SqliteSaver` 체크포인터, **디스코드 채널/스레드 ID = thread_id** (채널별 독립 맥락) |
| 장기 기억 | 옵시디언 볼트 RAG + 활동 온톨로지 YAML |
| 노션 (조직 게스트) | **내 개인 권한으로 Notion MCP/API 연결** → 봇이 개인 워크스페이스에 데일리루틴 템플릿을 채워두고, 내가 조직 페이지에 복사-붙여넣기만 |
| 블레이버스 (API 없음) | **알림 방식**: 정해진 시간에 봇이 "켰어요?" + 워크스페이스 링크를 디코로 전송 → 내가 클릭해서 시작/종료. 밤에 미종료 감지 잔소리 포함 |
| 인스타 | 반자동: 봇이 템플릿 캡션 + 이미지(텐미닛 캡처/타이머 사진)를 디코로 완성본 배달 → 복붙만 |
| 티스토리 | hermes RTF 클립보드 반자동 파이프라인 재활용 (봇이 HTML까지 준비) |
| 로컬 모델 | 파인튜닝→양자화→GGUF 모델을 1차 라우터로 (분류는 로컬 0원, 복잡한 판단은 Claude) |

```

   ├── ⏱️ 시간요원: 블레이버스 알림·미종료 감시 / 텐미닛 / 타이머 사진 OCR
   ├── 📢 발행요원: 티스토리 HTML 준비 / 인스타 캡션 배달 / study-log 확인
   └── 🧠 기억요원: RAG 검색 (옵시디언 + 대화기록)
   ▼
SQLite Checkpointer + 웹훅 페르소나 응답
```

## 3. 팔다리 8개 연결 방식

| # | 기능 | 등급 | 방식 |
|---|---|---|---|
| 1 | 텐미닛 | 🟢 자동 | 내가 만든 Vercel 앱 → API 라우트 추가 or DB 직접. task 기록 + year 색칠 |
| 2 | 옵시디언 | 🟢 자동 | 로컬 마크다운 직접 읽기/쓰기. TIL·코테노트 초안. RAG 원천 |
| 3 | 깃허브 study-log | 🟢 자동 | `gh` CLI로 Actions 상태 확인/트리거. "오늘 기록 올라갔어?" |
| 4 | 노션 데일리루틴 | 🟡 반자동 | 개인 워크스페이스에 MCP/API로 템플릿 채움 → 조직 페이지에 내가 복붙. 사진은 디코에 올린 걸 봇이 정리해 첨부 안내 |
| 5 | 블레이버스 | 🟡 알림 | cron: 아침 "시작하세요" + 링크 / 밤 "안 끄셨죠?" + 링크. 클릭은 내가 |
| 6 | 티스토리 | 🟡 반자동 | 옵시디언 글 → 봇이 HTML/RTF 준비(hermes 재활용) → 붙여넣기+발행만 내가 |
| 7 | 인스타 2계정 | 🟠 반자동 | 스윗포테이토(텐미닛 캡처) / 비니로그(타이머 사진) 캡션 템플릿 자동 완성 → 디코로 배달 |
| 8 | 오프라인 타이머 | 🟢 입력장치 | 사진을 디코에 던지면 Claude 비전으로 시간 판독 → 온톨로지 따라 다곳 전파 + 비니로그 캡션 생성 |

## 4. 킬러 기능: 활동 온톨로지

활동 1개가 기록처 4~5곳에 흩어지는 문제를 매핑 그래프로 해결.

```yaml
활동: 코딩테스트
  기록처:
    - 노션.데일리루틴.코딩테스트: 사진 첨부
    - 옵시디언: 코테 노트 (내 코드 + 모범 코드)
    - 텐미닛: task + timetable
    - 블레이버스: 시간 기록 (링크 알림)
    - study-log: 자동 반영 확인만
  트리거_문장: ["코테 끝", "백준 풀었어"]
```

→ 디코에 "코테 끝났어" + 사진 한 장이면 비서가 전파.
데일리루틴 6칸(코딩테스트/도착8시/운동/어드민나잇/영어스피킹/회고) 각각 온톨로지 항목으로 정의.

## 6. Streamlit — "공주 대시보드"

봇과 같은 SQLite를 읽는 조망용 화면 (디코=입력, Streamlit=조망):
- 오늘의 데일리루틴 6칸 체크 현황
- 블레이버스 주 100시간 진행률 게이지
- 텐미닛 year 히트맵 미러 + 인스타 발행 대기함
- RAG 검색창

## 7. 부가 기능

1. 아침 브리핑 (cron): 어제 빠진 칸 + 블레이버스 시작 링크
2. 저녁 마감 감사: hermes daily-closing 이식, crosscheck 잔소리
3. 주간 회고 자동 초안 → 옵시디언 저장
4. Claude API 비용 미터 (로컬 라우터 도입 효과 측정)
